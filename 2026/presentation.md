# ◈ CORE 2: SEMANTIC VIEWS

## The Missing Engram Layer

**PROBLEM:** Neural networks don't understand your business

```
USER: "What's our revenue?"
AI: ??? (Gross? Net? ARR? MRR? Which table? What filter?)
```

**SOLUTION:** Semantic Views encode business meaning

```yaml
# SEMANTIC VIEW DEFINITION
dimensions:
  - name: customer_tier
    synonyms: [customer type, tier, segment]
    description: Value classification (Gold/Silver/Bronze)
    
metrics:
  - name: total_revenue
    expression: SUM(amount) FILTER (WHERE status = 'COMPLETED')
    description: Total completed transaction revenue
```

# ◈ DEMO 2: SEMANTIC VIEW QUERIES

## Natural Language → Precise SQL

```bash
# CLI COMMAND
cortex analyst query "What is the average altitude of tracked flights?" \
  --view=DEMO.DEMO.ADSB_FLIGHT_TRACKING
```

**SEMANTIC VIEW METRICS:**
| Metric | Value | Description |
|--------|-------|-------------|
| `TOTAL_AIRCRAFT` | 26 | Distinct aircraft in grid |
| `TOTAL_FLIGHTS` | 21 | Active flight vectors |
| `AVG_ALTITUDE` | 19,137 ft | Mean cruising altitude |
| `AVG_GROUND_SPEED` | 330 kts | Mean velocity |

```sql
-- GENERATED SQL (via semantic view)
SELECT sv.total_aircraft, sv.avg_altitude, sv.avg_ground_speed
FROM SEMANTIC_VIEW(DEMO.DEMO.adsb_flight_tracking 
    METRICS total_aircraft, avg_altitude, avg_ground_speed) AS sv;
```
